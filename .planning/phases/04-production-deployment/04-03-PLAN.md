# Plan 04-03: Cloud Deployment — Neon + Render + Vercel

## Objective

Execute the actual cloud deployments for all three services. This plan is **not fully autonomous** — it requires the user to create accounts and configure services in cloud dashboards. It provides step-by-step instructions for each service.

---

## Pre-requisites

Before starting this plan, ensure Plans 04-01 and 04-02 are complete and committed to the `main` branch:
```bash
git add -A && git commit -m "feat(04): prepare backend and frontend for production"
git push origin main
```

---

## Tasks

### Task 1: Set up Neon PostgreSQL database

<read_first>
- `backend/database.py` — to confirm DATABASE_URL handling is correct (completed in Plan 04-01)
- `alembic/env.py` — to confirm it reads DATABASE_URL from env
</read_first>

<action>
**Step 1:** Create a Neon account at https://neon.tech (free — no credit card)

**Step 2:** Create a new project:
- Project name: `smart-crop`
- Region: Singapore (ap-southeast-1) — closest to India for lowest latency

**Step 3:** In the Neon dashboard → Connection Details → copy the connection string:
```
postgresql://USER:PASSWORD@HOSTNAME/DBNAME?sslmode=require
```

**Step 4:** Test the connection locally:
```bash
# Windows PowerShell
$env:DATABASE_URL = "postgresql://USER:PASSWORD@HOSTNAME/DBNAME?sslmode=require"
python run_backend_test.py
```
Expected: `Register successful: direct_test@test.com`

**Step 5:** Run Alembic migrations against Neon:
```bash
# Windows PowerShell
$env:DATABASE_URL = "postgresql://USER:PASSWORD@HOSTNAME/DBNAME?sslmode=require"
alembic upgrade head
```
If an `alembic/versions/` folder exists but is empty, first generate the initial migration:
```bash
alembic revision --autogenerate -m "initial_schema"
alembic upgrade head
```

**Step 6:** Verify in Neon console → SQL Editor:
```sql
SELECT table_name FROM information_schema.tables WHERE table_schema = 'public';
```
Expected: `users`, `scanhistory` (or similar) tables visible.
</action>

<acceptance_criteria>
- Neon project created and connection string obtained
- `alembic upgrade head` runs without errors against the Neon URL
- `SELECT * FROM users LIMIT 1` in Neon SQL Editor executes without error (empty result is fine)
</acceptance_criteria>

### Task 2: Deploy backend to Render

<read_first>
- `backend/requirements.txt` — confirm psycopg2-binary present
- `backend/main.py` — confirm CORS includes FRONTEND_URL env var
</read_first>

<action>
**Step 1:** Create a Render account at https://render.com (free — no credit card)

**Step 2:** Create a new Web Service:
- Connect to your GitHub repository
- Name: `smart-crop-backend`
- Region: Singapore (ap-southeast-1)
- Runtime: Python 3
- Root Directory: *(leave blank — repo root)*
- Build Command: `pip install -r backend/requirements.txt && cd backend && alembic upgrade head`
- Start Command: `uvicorn backend.main:app --host 0.0.0.0 --port $PORT`

**Step 3:** Add environment variables in Render dashboard → Environment tab:
```
DATABASE_URL = postgresql://USER:PASSWORD@HOSTNAME/DBNAME?sslmode=require
SECRET_KEY   = (generate: python -c "import secrets; print(secrets.token_hex(32))")
ALGORITHM    = HS256
```
Leave `FRONTEND_URL` blank for now — will update after Vercel deploy.

**Step 4:** Click "Create Web Service". Render will:
1. Clone the repo
2. Run the build command (installs deps + runs migrations)
3. Start the uvicorn server

**Step 5:** Wait for deploy to complete (~3-5 min). Then test:
```bash
curl https://smart-crop-backend.onrender.com/health
```
Expected: `{"status":"ok"}`

**Step 6:** Copy the Render URL (e.g., `https://smart-crop-backend.onrender.com`)
</action>

<acceptance_criteria>
- Render Web Service status shows "Live"
- `curl https://<render-url>/health` returns `{"status":"ok"}`
- Render deploy logs show no Python import errors
</acceptance_criteria>

### Task 3: Deploy frontend to Vercel

<read_first>
- `frontend/vercel.json` — confirm SPA rewrites exist (completed in Plan 04-02)
- `frontend/src/api/client.js` — confirm VITE_API_URL usage (completed in Plan 04-02)
</read_first>

<action>
**Step 1:** Create a Vercel account at https://vercel.com (free — no credit card, GitHub login works)

**Step 2:** Import your GitHub repository in Vercel:
- Root Directory: `frontend`
- Framework Preset: Vite (auto-detected)
- Build Command: `npm run build`
- Output Directory: `dist`

**Step 3:** Add environment variables in Vercel project settings → Environment Variables → Production:
```
VITE_API_URL = https://smart-crop-backend.onrender.com
```
*(Use the Render URL from Task 2)*

**Step 4:** Click "Deploy". Vercel will:
1. Install deps (`npm install`)
2. Build the Vite app (`npm run build`)
3. Deploy static files to global CDN

**Step 5:** Wait for deploy (~2 min). Then test:
- Navigate to `https://smart-crop-disease.vercel.app` (or your assigned URL)
- Confirm the app loads (Register/Login buttons visible)
- Navigate directly to `https://<vercel-url>/history` — should NOT 404

**Step 6:** Copy the Vercel URL (e.g., `https://smart-crop-disease.vercel.app`)
</action>

<acceptance_criteria>
- Vercel deployment status shows "Ready"
- App loads at Vercel URL (no blank screen)
- `/login` route renders login form
- Direct-access URL like `<vercel-url>/history` returns status 200 (not 404)
</acceptance_criteria>

### Task 4: Update CORS and finalize cross-origin connection

<read_first>
- `backend/main.py` — current FRONTEND_URL env var CORS configuration
</read_first>

<action>
**Step 1:** Go to Render dashboard → smart-crop-backend → Environment Variables

**Step 2:** Add:
```
FRONTEND_URL = https://smart-crop-disease.vercel.app
```
*(Use your actual Vercel URL from Task 3)*

**Step 3:** Render auto-redeploys on env var change. Wait ~1-2 min for the redeploy.

**Step 4:** Run end-to-end test from the browser:
1. Open `https://<vercel-url>/register`
2. Register with a new email/password
3. After redirect to login, login with same credentials
4. Confirm redirect to dashboard (`/`)
5. Open DevTools → Network tab → confirm `/api/auth/register` returned 200 (not CORS error)

**Step 5:** Write the deployed URLs to `README.md`:
```markdown
## 🌐 Live Demo

- **Frontend:** https://smart-crop-disease.vercel.app
- **Backend API:** https://smart-crop-backend.onrender.com
- **API Docs:** https://smart-crop-backend.onrender.com/docs
```
</action>

<acceptance_criteria>
- `FRONTEND_URL` set in Render env vars to the Vercel URL
- Registration via the Vercel frontend works without CORS errors (Network tab shows 200)
- Login after registration succeeds (redirects to dashboard)
- `README.md` contains the deployed URLs
</acceptance_criteria>
